Introducción

 

A partir de la revisión de los materiales sugeridos en la plataforma y una vez identificada durante la primera etapa del proyecto integrador la problemática de una organización en la que se pueda intervenir, resolviendo el problema por medio de la implementación un servicio web; en esta segunda etapa procederemos a detallar los puntos importantes para el desarrollo, donde se definen los datos, los medios de transformación e integración de los mismos, así como también la infraestructura necesario para el transporte y almacenamiento de datos; por ultimo también se describirá la interfaz de los usuarios.

 

ETAPA 1

 

Descripción de la organización (giro sector que atiende, posición en el mercado)

Empresa Internacional de producción y venta de gorras y prendas deportivas con presencia mundial, actualmente abarca distintos deportes como Futbol Americano, Béisbol, Basquetbol y Futbol Soccer principalmente.

Impacto de los servicios web en la organización

Actualmente el uso de los servicios web esta enfocado en la correcta distribución y recepción de producto con las distintas empresas y clientes, lo cual nos da un control de distribución contra pagos. De esta manera las empresas tienen un sistema seguro para tener un control interno de mercancía al mismo tiempo que permite el flujo correcto de la información fiscal, facilitando procesos contables.

Descripción de la situación o problema a intervenir a través del establecimiento de un servicio web

Actualmente la empresa tiene un buen control de que las Ordenes de Compra se están distribuyendo de forma correcta con los clientes.

El proceso para realizar estas ordenes es mediante una Pre Compra para producción, la cual sufre varios cambios durante el proceso desde que es emitida hasta la entrega a cliente, lo cual no permite medir la ejecución real desde el inicio del proceso al final. Esto ha generado desconfianza por parte de los clientes ya que ellos, esperan un número de acuerdo con su compra inicial, y al final con los ajustes no se recibe la cantidad acordada.

El desarrollo de este servicio web, permitirá medir la ejecución de lo planeado vs lo real, midiendo al mismo tiempo las ventas y poder tener datos que nos permitan generar mejores algoritmos para ir mejorando el proceso de Pre Compra y se reduzcan las devoluciones de producto a nuestra empresa.

Descripción de la metodología o pasos generales en el desarrollo de un servicio web

Como primer paso se realiza un análisis del problema que se busca resolver.

Determinado el problema, se debe de acotar los alcances del proyecto y los resultados esperados.

Se debe de analizar la estructura de los involucrados para determinar las herramientas adecuadas para el desarrollo.

Se plantea la arquitectura y el flujo de la información, así como los datos que se deben de enviar y los que se esperan recibir para cumplir con los requerimientos mínimos.

Se crea una matriz de pruebas, donde se deben de plasmar todos los escenarios posibles que pudieran presentarse dentro del entorno donde se desarrollara la aplicación.

Corrección de errores y ajustes de planteamiento principal.

Se evalúan los resultados y se definen el proceso para la implementación.

 

Justificación de la arquitectura a utilizar de acuerdo a los requerimientos de la organización (SOAP, REST o ambas)

Para este caso la opción es el uso del protocolo SOAP, el cual ya esta siendo utilizado actualmente dentro de los servicios web existentes. Lo cual permite reutilizar esta estructura y no requerirá grandes modificaciones en los sistemas de seguridad por ambas partes, dando mayor confianza y velocidad para los desarrolladores.

 

ETAPA 2

 

Definición de fuentes de datos a utilizar

Debido a que los datos provienen de distintas fuentes, ya que cada uno de los clientes tienen un sistema de datos distinto, se propone la creación de un Layout estándar en el que se puedan basar para que el envío de información se integre de forma correcta dentro de nuestro sistema, fijando datos mandatorios y opcionales para lograr análisis más detallados siempre y cuando la base de datos original lo permita.

Se dividirán en tres grupos:

Preventa: que contendrá la información de la orden de compra original

Ordenes de Compra: que incluye cada una de las entregas al cliente relacionadas a la Orden de compra original.

Ventas: Envía el detalle de ventas de los productos para evaluar su comportamiento.

Procesos de extracción, transformación y carga de datos

Extracción:

La información será extraída de los RP de los clientes con un desarrollo interno del cliente, dando libertad de código a la empresa, se define un archivo de texto plano usando como separador “||” teniendo en la primera línea los encabezados donde se identificarán los datos principales del cliente, así como los identificadores de documento y sus relacionados. Posteriormente vendrán los detalles de los productos con sus atributos principales.

Transformación:

A partir del texto plano se genera el código conforme al layout definido, respetando el orden, tipo de dato, tamaño y separadores se transforman en código fuente para realizar la inserción en las distintas tablas en la base de datos.

 

Carga de datos:

La inserción en la base de datos a través del script que transforma los archivos de texto plano, en las tablas de la base de datos. Durante el proceso de carga se deben de realizar validaciones de la integración ya que es necesario validar que la información este completa de acuerdo con el layout, que los tipos de datos correspondan y evitar que la información sea duplicada en la base de datos para asegurar que la información sea precisa.

Mecanismos e infraestructura para el almacenamiento y gestión de datos

Se contempla el uso de una base de datos Oracle, la cual se usa actualmente solo para almacenar los documentos provenientes el punto de venta de las tiendas propias. Este servidor se encuentra en los servidores virtuales contratados en la solución de Microsoft Azure. De esta manera aseguramos la disponibilidad del servicio, haciendo uso de la seguridad y utilidades de respaldos para mantener la base de datos segura.

Interfaz de usuario a utilizar

Cliente: Servicio donde pueden validar las cargas realizadas al sistema, donde pueden recibir retroalimentación de los errores producidos, o en su defecto la confirmación de la integración de la información.

Interno empresa: Visualización de las cargas y errores de esta. Se planea una interfaz visual para validar los documentos y su contenido detallado. Se generan reportes diarios sobre el estatus para alertar la falta de información, así como también satisfacer el principal objetivo del desarrollo para tener una medición de la Orden de Compra Original contra la ejecución, para verificar el producto se esté moviendo de acuerdo con la planeación, esto permite generar alertas anticipadas y permita reaccionar a tiempo y que además se pueda medir el desempeño de los responsables de las cuentas.

Mecanismos de seguridad

Se protege la información de punto a punto por medio de la transferencia SFTP. El acceso a la interfaz de confirmaciones esta cifrado y se definen roles para determinar los accesos de acuerdo con las necesidades de cada rol.

En la infraestructura con los servicios de seguridad propios de Azure y Oracle para proteger la aplicación y la integridad de los datos.

 

 



 

 

 

Conclusión:

Dentro la primera etapa, nos enfocamos en un problema en el cual podamos contribuir a su solución con la creación de un servicio web, en este caso se desea contar con una base de datos sólida, poder ofrecer a mejores propuestas de producto, reduciendo también el impacto que genera las devoluciones de producto a nuestra empresa, lo cual se traduce en pérdidas de capital.

En esta segunda etapa, se realiza un análisis mas a fondo de la solución. En esta parte identifiqué que para poder tener una comunicación de los datos efectiva a través de las distintas fuentes de información es necesario establecer un formato general, donde se cuente con la información mínima necesaria para contar con los datos necesarios para efectuar los distintos análisis requeridos para evaluar el comportamiento entre la ejecución de la planeación general entre los clientes y otorgando herramientas a los ejecutivos de cuenta para tener el producto en el lugar y momento adecuado para tener un rendimiento de operación optimo.

De igual manera se revisan los recursos con los que la empresa cuenta actualmente para evitar duplicar información o procesos y tener soluciones integradas para no tener distintos puntos de fallo, y tener una base de conocimientos unificada y con la que estemos familiarizados. Se busca que los medios existentes cuenten con la seguridad necesaria para tener un correcto resguardo de los datos y al mismo tiempo un fácil acceso a ellos, para explotar al máximo el poder la información.

Contar con estos datos oportunamente contribuye en la toma de decisiones en las distintas áreas, así como dar datos duros para justificar las decisiones tomadas y no sólo basarse en la experiencia, que si bien es de gran importancia, tener el respaldo de datos da mayor peso a estas.
